⚡ Кастомные действия

Узнайте, как создавать и регистрировать кастомные действия для ваших Telegram-ботов.

Что такое кастомные действия?

Кастомные действия позволяют расширить Scenarium собственной функциональностью. Вы можете создавать действия, которые выполняют специфические задачи, не покрытые встроенными действиями.

Создание кастомного действия

Для создания кастомного действия необходимо реализовать интерфейс ICustomAction:

Интерфейс кастомного действия

interface ICustomAction {
  name: string;
  execute(context: ProcessingContext): Promise;
}

Пример: Действие погоды

Реализация действия погоды

import { ICustomAction, ProcessingContext } from 'scenarium';

export class WeatherAction implements ICustomAction {
  name = 'Weather';
  
  async execute(context: ProcessingContext): Promise {
    const city = context.localContext.data.city || 'Moscow';
    const weather = await this.getWeather(city);
    
    await context.bot.sendMessage(context.userId, 
      `Погода в ${city}: ${weather.temperature}°C, ${weather.description}`
    );
  }
  
  private async getWeather(city: string) {
    // Ваша логика API погоды здесь
    return {
      temperature: 20,
      description: 'Солнечно'
    };
  }
}

Регистрация кастомных действий

Используйте RegistryManager для регистрации ваших кастомных действий:

Пример регистрации

import { RegistryManager } from 'scenarium';
import { WeatherAction } from './WeatherAction';

// Регистрируем кастомное действие
RegistryManager.registerAction(new WeatherAction());

Использование кастомных действий в сценариях

После регистрации вы можете использовать ваше кастомное действие в сценариях:

Сценарий с кастомным действием

{
  "menuItems": {
    "Weather": {
      "onNavigation": {
        "action": "Weather",
        "city": "London"
      }
    }
  }
}

Лучшие практики

  • Используйте описательные имена для ваших действий
  • Обрабатывайте ошибки корректно
  • Используйте объект контекста для доступа к данным пользователя и экземпляру бота
  • Сосредоточьте действия на одной ответственности
