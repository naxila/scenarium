RequestInput is an action for requesting text input from the user. Supports hints, result processing, input cancellation and behavior configuration.

üí° Usage Examples

Simple name request

{
  "action": "RequestInput",
  "key": "userName",
  "hint": "What is your name?",
  "onDone": {
    "action": "SendMessage",
    "text": "Nice to meet you, {{input.userName}}!"
  }
}

Request with cancellation

{
  "action": "RequestInput",
  "key": "email",
  "hint": "Enter your email:",
  "onDone": {
    "action": "SendMessage",
    "text": "Email saved: {{input.email}}"
  },
  "onCancel": {
    "action": "SendMessage",
    "text": "Email input cancelled"
  },
  "cancelText": "‚ùå Cancel"
}

Request with storage saving

{
  "action": "RequestInput",
  "key": "phone",
  "hint": "Enter phone number:",
  "onDone": {
    "action": "Store",
    "key": "userPhone",
    "value": "{{input.phone}}"
  },
  "onCancel": {
    "action": "Navigate",
    "menuItem": "Main"
  }
}

{
  "action": "RequestInput",
  "key": "tempData",
  "hint": "Enter temporary data:",

  "onDone": {
    "action": "SendMessage",
    "text": "Data processed: {{input.tempData}}"
  }

  }

üîÑ How it works

Input waiting state

When executing RequestInput, the system:

  • Saves input waiting state in user session
  • Sends hint with cancel button (if configured)
  • Puts bot in text input waiting mode
  • Blocks processing of other actions until input is received

Input processing

When receiving text from user:

  • Saved in {{input.key}} where key is the specified key
  • Executes onDone action if configured
  • Clears input waiting state
  • When clearInputOnDone: true input is removed from session

  • {{input.phone}} - Entered phone

  • {{userContext.currentMenu}} - Current menu
  • {{userContext.data}} - User data

‚ö†Ô∏è Important notes

  • Key required: Parameter key must be a string
  • Single waiting: Only one input waiting can be active at a time
  • Input cancellation: New RequestInput cancels the previous one
  • Timeout: No automatic timeout, user must enter text or cancel
  • Validation: No built-in validation, checking should be in onDone
