üîß How it works

State clearing

When executing CancelAwaitingInput, the system:

  • Clears input waiting state in user session
  • Removes data associated with waiting
  • Returns bot to normal operation mode
  • Allows processing new commands and actions

When to use

CancelAwaitingInput is useful in cases:

  • User doesn't respond for a long time
  • An error occurred during waiting
  • User selected cancel action
  • Need to forcibly complete waiting

  • Security: Action executes only if there's active input waiting
  • State: After cancellation, all waiting data is removed
  • Reuse: Can be used multiple times in a row
  • Performance: Action executes instantly
