üîÑ How it works

Back Stack

The back stack allows users to return to previous menus. With each navigation where addToBackStack: true, the current menu is added to the stack.

Stack addition rules:

  • If uniqueInStack: true - all previous occurrences of this menu are removed
  • If uniqueInStack: false - menu is added even if it already exists in the stack
  • Maximum stack size: 20 elements
  • When limit is exceeded, the oldest element is removed

üóëÔ∏è Message Management

Deleting Previous Messages

When removePreviousMessage: true, the system:

  • Deletes the previous message from Telegram
  • Clears associated inline buttons
  • Resets the last message ID

‚ö†Ô∏è Important notes

  • Menu existence: If the specified menu item doesn't exist, fallback to onStartActions is executed
  • Back stack: On fallback, the stack is completely cleared
  • Message deletion: Cannot delete messages older than 48 hours
  • Performance: Frequent message deletion may slow down bot performance
  • Action clearing: When deleting messages, all associated inline actions are cleared
