üîÑ How it works

Navigation stack

When executing Back, the system:

  • Checks for elements in the navigation stack
  • If stack contains more than 1 element - removes the last one
  • Navigates to the previous menu item
  • If stack is empty or contains only 1 element - executes fallback

Fallback to initial actions

When there are no elements in the stack:

  • Current menu and stack are cleared
  • Actions from onStartActions are executed
  • If onStartActions are missing - return message is sent

‚ö†Ô∏è Important notes

  • Navigation stack: Must contain at least 2 elements for return
  • Delete messages: Cannot delete messages older than 48 hours
  • Fallback: When stack is empty, return to beginning is executed
  • Performance: Frequent message deletion may slow down performance
