üí° Usage Examples

Simple custom action

{
  "action": "CustomAction",
  "customLogic": "my-custom-handler",
  "data": {
    "param1": "value1",
    "param2": "value2"

Action with interpolation

{
  "action": "CustomAction",
  "handler": "processUserData",
  "userData": {
    "name": "{{userContext.data.name}}",
    "email": "{{userContext.data.email}}"
  },
  "options": {
    "validate": true,
    "save": false
  }
}

Action with function

{
  "action": "CustomAction",
          "handler": "calculateResult",
          "input": {
            "function": "JoinToString",
            "values": ["{{input.value1}}", "{{input.value2}}"]
          },
          "operation": "multiply"
        }

üîß How it works

Action processing

When executing CustomAction, the system:

  • Interpolates all parameters
  • Logs action to console
  • Passes control to custom handler
  • Updates user activity

Functionality extension

CustomAction allows:

  • Create custom handlers
  • Integrate with external APIs
  • Implement complex business logic
  • Add new features without changing core

  • Handlers: Require implementation in bot code
  • Performance: Depends on handler implementation
  • Errors: Handled in custom code
  • Compatibility: Must be compatible with system
