Function for debugging and outputting objects in various formats with console logging.

üìã Description

Dump is a debugging function that takes any object and returns its string representation in the selected format. Always outputs information to the console for debugging.

üéØ Purpose

  • Debugging variables and objects
  • Logging data to console
  • Formatted output of complex structures
  • Data type analysis

Parameter Type Required Default Description
value any ‚úÖ - Object to debug (can be variable, function, or string with interpolation)
format string ‚ùå 'json' Output format: 'json', 'compact', 'table', 'inspect', 'type'
depth number ‚ùå 2 Nesting depth for 'inspect' format

üìä Output Formats

Available formats

  • json - Pretty JSON formatting with indentation
  • compact - Compressed JSON without indentation
  • table - Tabular representation for object arrays
  • inspect - Detailed inspection with limited depth
  • type - Data type information

üîß Value Processing

Value types

The function processes various value types:

  • Strings with interpolation: Interpolated before debugging
  • Functions: Executed and result is debugged
  • Objects: Serialized in selected format
  • Arrays: Processed as objects or tables
  • Primitives: Converted to strings

Console logging

The function always outputs to console:

  • Original value
  • Data type
  • Additional information (array length, object keys)
  • Formatting result
